## How to Contribute

### Quick Start

1. **Fork this repository** to your GitHub account
2. **Create a branch** for your contribution: `git checkout -b my-contribution`
3. **Make your changes** following the templates and guidelines
4. **Submit a Pull Request** with clear description
5. **Engage with reviewers** during the review process

### Alternative Methods

If you're not comfortable with GitHub:
- **Email submission**: Send to hello@soullab.org
- **Issue creation**: Open a GitHub Issue labeled "Contribution Submission"
- **Discussion forum**: Post in GitHub Discussions

## Submission Guidelines

### Before You Submit

**1. Use the appropriate template**:
- Case studies: Use case template
- Protocols: Use protocol template
- Research: Follow academic formatting
- Code: Follow repository structure

**2. Ensure quality**:
- Proofread for clarity and grammar
- Check all links work
- Verify anonymization (for cases)
- Test protocols (minimum 5 applications)
- Run code tests (for technical contributions)

**3. Review ethics**:
- Client confidentiality protected
- No identifying information
- Consent obtained if required by your jurisdiction
- No harm to clients from sharing
- Within your scope of practice

**4. Attribute sources**:
- Cite Jung, Edinger, Hillman where relevant
- Credit colleagues whose ideas you're building on
- Link to related work in Commons
- Don't plagiarize

### During Submission

**Pull Request description should include**:
- **Type of contribution**: Case study, protocol, research, etc.
- **Brief summary**: 2-3 sentences describing the contribution
- **Why it matters**: What does this add to the Commons?
- **Testing/validation**: How has this been tested or verified?
- **Related work**: Links to related Commons content

**Example PR description**:
```markdown
## Type: Clinical Protocol

## Summary
Submitting a protocol for working with Puer-Senex splits in clients oscillating
between inflation (eternal youth) and deflation (premature old age).

## Why it matters
This signature appears frequently in practice but wasn't included in core suite.
Fills a gap in operation-specific protocols.

## Testing
Applied with 8 clients over 14 months. Average coherence increase: +0.22.
Detailed outcomes included in protocol.

## Related work
- Builds on Separatio Protocol
- Complements Nigredo-Rubedo Oscillation case study
```

## Review Process

### What to Expect

**1. Initial Screening** (2-3 days)
- **Moderator review**: Checks for appropriate content, template usage, anonymization
- **Outcome**: Approved for peer review, or feedback for revisions

**2. Peer Review** (1-2 weeks)
- **Two practitioners/researchers** review for:
  - Clinical soundness (for cases/protocols)
  - Theoretical accuracy (Jung-Edinger-Hillman alignment)
  - Practical usefulness (can others apply this?)
  - Ethical compliance
- **Outcome**: Approved, approved with minor revisions, major revisions needed, or declined

**3. Revisions** (if needed)
- Reviewers provide specific feedback
- You make changes and resubmit
- Reviewers confirm revisions address concerns

**4. Publication**
- Approved contributions merged into Commons
- Your name/pseudonym credited
- Community can now access, use, and build on your work

### Review Criteria

**For Case Studies**:
- [ ] Thoroughly anonymized (client unidentifiable)
- [ ] Spiralogic framework applied appropriately
- [ ] Clear progression through transformation
- [ ] Honest about challenges and limitations
- [ ] Useful for other practitioners

**For Clinical Protocols**:
- [ ] Evidence of real-world testing
- [ ] Theoretically grounded in Jung-Edinger-Hillman
- [ ] Clear, actionable steps
- [ ] Safety protocols specified
- [ ] Includes contraindications and risks

**For Research**:
- [ ] Methodology sound
- [ ] Results clearly reported
- [ ] Limitations acknowledged
- [ ] Contributes new knowledge
- [ ] Ethical approval obtained (if required)

**For Technical Code**:
- [ ] Code works (tests pass)
- [ ] Follows repository conventions
- [ ] Documented clearly
- [ ] Doesn't break existing features
- [ ] Serves therapeutic goals (not just technical)

### If Your Contribution is Declined
