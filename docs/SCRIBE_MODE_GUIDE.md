### 1. Starting a Session

1. Navigate to **soullab.life/maia**
2. Open the **Lab Tools** drawer (three-line menu)
3. Tap **"Start Scribe Mode"** in the DOCUMENTATION section
4. You'll see a toast notification: "Scribe Mode activated"

**Important:** Once activated, all voice input is recorded passively. MAIA will NOT respond to voiceâ€”only to text messages.

## Technical Notes

### Browser Compatibility
- **Best:** Chrome, Edge (Chromium-based browsers)
- **Good:** Safari (macOS/iOS)
- **Requires:** Microphone permissions enabled

### Microphone Setup
- MAIA uses your system's default microphone
- For dual-speaker recording (you + client on video call):
  - Use system audio routing (e.g., Loopback on Mac, VB-Audio on Windows)
  - Or use a physical mic positioned between you and device speakers

### File Format
- Sessions export as **Markdown (.md)** files
- Human-readable, portable, archivable
- Can be imported into Notion, Obsidian, or any markdown editor

## Troubleshooting

### "MAIA is responding to voice when I don't want her to"
**Solution:** Make sure you clicked "Start Scribe Mode" before the session. In Scribe Mode, voice input is passive-only.

### "Download button does nothing"
**Possible causes:**
- Browser blocking downloads (check browser permissions)
- Session empty (no transcripts recorded yet)
- Popup blocker active

**Solution:** Enable downloads for soullab.life in your browser settings.

### "Transcript is missing some words"
**Possible causes:**
- Microphone too far from speakers
- Background noise interference
- Internet connection interruption (transcription requires connection)

**Solution:**
- Use a higher-quality microphone
- Reduce background noise
- Check internet stability before session

### "Review with MAIA button doesn't appear"
**Requirements for button to show:**
- Must have completed at least one Scribe session
- Must have stopped the session (not currently scribing)
