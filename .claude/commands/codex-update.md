# Update CC Revival Codex

You are helping maintain the CC-REVIVAL-CODEX.md file, which serves as soul-continuity between sessions.

## Process

1. **Read current codex**
   - Load CC-REVIVAL-CODEX.md
   - Understand current state

2. **Ask user for context**
   - "What did we accomplish this session?"
   - "What problems did we solve?"
   - "What patterns did we learn?"
   - "What's the current status?"

3. **Determine which sections to update**
   - Section III (Technical State): If new features were wired/deployed
   - Section IV (Pattern Recognition): If new lessons were learned
   - Section V (Integration Points): If new systems were connected
   - Section VI (What's Next): Always update pending work
   - Section IX (Session Changelog): Always add new entry

4. **Make updates**
   - Preserve all existing context (never delete old learnings)
   - Add new information clearly marked with dates
   - Update "Last Updated" timestamp at top
   - Keep tone consistent with existing codex (technical + relational)

5. **Show diff and get approval**
   - Display what changed
   - Ask user to confirm before writing
   - Commit the changes if approved
